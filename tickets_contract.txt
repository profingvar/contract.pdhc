--- Ticket 1 Results (derived from provider.pdhc/docs/data_package_reference.md, Section 8) ---

1.1 [ ] FHIR term[] validation — extend ensure_contract_shape() in fhir.py
   - Accept and validate term[] array inside the FHIR Contract resource
   - Two term types: "request_scope" (outbound concepts) and "return_scope" (return concepts)
   - request_scope contains asset[] with type "outbound_concept" and typeReference[] pointing to plan.pdhc concept URLs
   - return_scope contains asset[] with types "obligatory_return" and "optional_return", each with typeReference[]
   - Validate that typeReference URLs match the pattern https://plan.pdhc.se/api/v1/concepts/<guid>
   - Backward compatible: term[] is optional — contracts without it permit all concepts

1.2 [ ] Scope extraction helper — new function get_contract_scope(contract_guid)
   - Extract permitted concept GUIDs from term[].asset[].typeReference[]
   - Return structured result: { request_scope: [concept_guids], return_scope: { obligatory: [concept_guids], optional: [concept_guids] } }
   - Return None/null if no scope terms defined (backward compatible — means "all permitted")
   - Location: fhir.py or new scope_service.py

1.3 [ ] New endpoint — GET /fhir/Contract/<guid>/scope
   - Lightweight endpoint returning only the concept scope + contract status
   - Response: { contract_guid, status, scope_defined, request_scope: [...], return_scope: { obligatory_return: [...], optional_return: [...] } }
   - When no scope defined: scope_defined=false, both lists null
   - When contract revoked/terminated: return empty lists (gateway rejects all submissions)
   - Auth: public + rate-limited (same as GET /fhir/Contract/<guid>)
   - This is the endpoint gateway.pdhc will call to validate provider submissions against contract

1.4 [ ] Internal service key auth — add X-Service-Key support
   - Add INTERNAL_SERVICE_KEY to config.py
   - Add middleware/decorator that validates X-Service-Key header for internal endpoints
   - Gateway.pdhc will use this key to call the scope endpoint without SSO
   - The scope endpoint (1.3) should accept either public rate-limited access OR X-Service-Key (internal callers bypass rate limit)

1.5 [ ] Scope editor in contract admin UI
   - Add a scope section to the contract create/edit form
   - Search/select concepts from plan.pdhc vocabulary API (GET https://plan.pdhc.se/api/v1/concepts?search=...)
   - Separate lists for: outbound concepts (request_scope) and return concepts (obligatory + optional)
   - On save, inject the term[] structure into the FHIR Contract JSON before storing
   - Show current scope on contract detail view
   - Can be deferred if no admin UI exists yet

1.6 [ ] Tests for all scope reforms
   - FHIR validation: term[] accepted, bad term types rejected, missing typeReference rejected
   - Scope extraction: returns correct concept lists, handles missing terms, handles empty terms
   - Scope endpoint: returns scope for scoped contract, returns scope_defined=false for legacy contract, returns empty for revoked contract
   - Internal service key: valid key accepted, missing/invalid key rejected
